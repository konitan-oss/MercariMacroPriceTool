## 目的
- 本日完走済みスキップを個別に解除できるよう、選択行の RunCount/LastRunDate をリセットする機能を追加。
- FetchItemsButton_Click line 119 で発生していた NullReference を再発しないよう防御（原因：CollectionView/Filter 実行時の null 前提不足）。

## 実施内容
- Storage: ResetItemAsync を追加し、RunCount を任意値に更新・LastRunDate を NULL にする API を提供。
- App UI: MainWindow に「選択商品をリセット」ボタンを追加、DataGrid に x:Name を付与して選択行を特定。
- App ロジック: ResetItemButton_Click で確認ダイアログ→DB更新→RunCount を 0 に反映、対象なし/例外もログ化し証跡保存をベストエフォート。
- NullReference 対策: ApplySettingsToUi 以降での新規処理でも CollectionView/SelectedItem null を防御し、例外はログ＋続行。
