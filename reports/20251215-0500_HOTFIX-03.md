1 目的  
EditClick の CSS セレクタにコメント文字が混入し「Unexpected token #」で停止する問題を解消し、候補試行とログを健全化する。

2 実施内容  
- docs/SELECTORS.md をコメントなしの純粋なセレクタ列に整理し、編集ボタン候補をリンク優先で明記。  
- SelectorsConfig で「#」以降を切り捨てる読み込み処理を維持しつつ、デフォルト候補をコメントなしで定義。  
- MercariAutomationService の EditClick 処理を複数候補順次試行＋リトライ付きで実装し、セレクタ/成功ログが純粋な文字列になるよう修正。NavigateItem は DOMContentLoaded＋90s/ main#main 待ちのまま。  
- MainWindow / ListingRow の文字列化けをリセットし、ビルド可能な状態へ再整備（ログ文言含む）。

3 変更ファイル一覧  
- docs/SELECTORS.md  
- src/MercariMacroPriceTool.Automation/SelectorsConfig.cs  
- src/MercariMacroPriceTool.Automation/MercariAutomationService.cs  
- src/MercariMacroPriceTool.App/MainWindow.xaml.cs  
- src/MercariMacroPriceTool.App/ListingRow.cs

4 確認手順（コニシがやる手順）  
- `dotnet build MercariMacroPriceTool.sln` を実行し成功することを確認。  
- アプリを起動し、価格改定を1件実行。ログに EditClick の候補試行が純粋なセレクタで出ること、`Unexpected token #` が発生しないことを確認。  
- 編集ボタンがリンク表示の場合でも候補を順次試し、次のステップへ進むことをログで確認。

5 既知の課題と次の一手  
- メルカリ側DOM変化でセレクタが変わる可能性は残る。反応が悪い場合は docs/SELECTORS.md の候補を追加・更新する。  
- Navigate 以外のステップでも稀にタイムアウトすることがあるため、必要に応じて待機設定の調整や evidence 取得内容の確認を継続。  
- 実機での複数商品実行動作は未再確認のため、次回運用時にログを確認しながら追従。
