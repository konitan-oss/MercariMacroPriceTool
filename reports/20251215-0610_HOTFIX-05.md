1 目的  
編集画面で「更新/変更する」待ちによるタイムアウトを避け、停止/再開ボタンで価格変更を確定できるようにする。ポップアップに遮られても自動で閉じつつクリックを完了させる。

2 実施内容  
- docs/SELECTORS.md に Pause/Resume 用セレクタ候補を追加（停止・再開で保存確定する前提）。  
- SelectorsConfig に PauseSelectors/ResumeSelectors を追加し、PopupCloseSelectors を維持。  
- MercariAutomationService の価格改定フローを Save 工程なしに再構成（Edit→PriceInput→Pause→待機→Edit→PriceInput復帰→Resume→待機）。Pause/Resume 後に各状態ボタンの出現を待って成功判定。  
- クリック共通処理を強化：通常クリック→ポップアップ閉鎖→再クリック→JSクリックの順でリカバリ。  
- RUNBOOK に「更新ボタンは使わず停止/再開で確定する」ことと、ポップアップが閉じられない場合の暫定手順を追記。

3 変更ファイル  
- docs/SELECTORS.md  
- docs/RUNBOOK.md  
- src/MercariMacroPriceTool.Automation/SelectorsConfig.cs  
- src/MercariMacroPriceTool.Automation/MercariAutomationService.cs

4 確認手順（コニシさん）  
- `dotnet build MercariMacroPriceTool.sln` が成功すること。  
- アプリ起動→1件チェック→「価格改定を実行」。ログで PriceInput success → Pause success（待機①へ）→ Resume success（待機②へ）を確認。  
- ログに Save/更新待ちのタイムアウトが出ず、ポップアップ出現時は `[PopupDismiss]` ログとクリックリカバリが出ることを確認。

5 動作確認結果  
- `dotnet build MercariMacroPriceTool.sln` 成功。  
- Save 待ちを行わないフローに切り替わり、ポップアップ遮断時は閉鎖＋再クリック＋JSクリックで継続できるログを確認（実機確認時にさらに追跡予定）。

6 既知の課題  
- メルカリ側の UI 変化で Pause/Resume ボタンやポップアップが変わる可能性があるため、SELECTORS.md の候補追加で追随が必要。  
- 長時間残る別種のオーバーレイがある場合は evidence を参照し、セレクタや閉鎖ロジックの追加を検討。  
- Save ボタン経由の確認は行っていないため、万一停止/再開で確定しない画面変更があれば再度要調整。
